#UDP

  • 基于IP协议
  • 对IP的简单包装：复用/分用，简单的错误校验
  • “Best effort”服务，可能丢失，非按序到达
  • 无连接，每个UDP段的处理独立于其他段

#滑动窗口协议

  • 窗口大小为N，最多允许N的分组未确认

  • n号分组timeout时

    • GBN（Go-Back-N）
      • 累计确认机制
      • 重传大于等于n的所有未收到ack的所有分组
      • 对于乱序到达的分组，不期望的分组直接丢弃
    • 选择重传（selective repeat）
      • 单独确认
      • 缓存乱序到达的分组
      • 接收方也有相应的滑动窗口
      • 窗口大小需要小于等于序号空间的一半

#网络层

  • 转发
    • 输入端口到输出端口
    • 最长前缀匹配规则

  • 路由

    • 确定转发路径

  • 分片与重组

    • 路由器只分片不重组，重组任务留给接收方。

  • IP地址

    • A类：0开头，高8位网络号，低24位主机号
    • B类：10开头，高16位网络号
    • C类：110开头，高24位网络号
    • D类：1110开头，用于多播
    • E类：1111开头，保留用作研究

  • 子网和子网掩码

    • 子网中不能分配给主机的是主机号全0或全1。
    • 主机号全0代表子网，全1代表广播地址。

  • CIDR（Classless InterDomain Routing，无类域间路由）

    • a.b.c.d/x（x为前缀长度）
    • 路由聚集

  • 如何获取IP地址

    • 1、硬编码
      • 静态配置
    • 2、DHCP协议（工作在应用层）
      • 从DHCP服务器动态获取
      • 允许IP地址重用

  • NAT

    • 所有离开本地网络去往Internet的数据报的源IP地址替换成相同的IP地址和不同的端口号

  • ICMP

    • 差错报告
      • 目的不可达
      • 源抑制
      • 超时/超期，一般是TTL
      • 参数错误问题
      • 重定向
    • 网络探寻
      • Echo请求与应答
      • 时间戳的请求与应答报文

  • IPv6

  • 路由算法

    • 链路状态
      • 存在振荡，不可容忍。
    • 距离向量
      • 好消息传播快
      • 坏消息传播慢
        • 通过毒性逆转技术避免

  • 层次化路由

    • 自制系统内（IGP）

      • RIP（一个运行在UDP之上的应用层协议）
      • OSPF：（安全、多条费用相等的路径）

    • 自制系统间（BGP）
